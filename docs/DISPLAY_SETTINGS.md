# OLED 디스플레이 밝기·수명 설정 (SSD1322)

AR22 LCU의 OLED(SSD1322, 256×64) 밝기 설정 기준 문서.
**가독성 유지 + 수명(휘도 감쇠·번인) 연장** 균형을 목표로 한다.

## 밝기 제어 구조 (3종)
| 제어 | 명령 | 범위 | 역할 |
|---|---|---|---|
| Set Contrast Current | `0xC1` | 0~255 | 세그먼트 구동 전류(전체 밝기). 전류↓ → 수명↑ |
| Master Contrast Current | `0xC7` | 0~15 | 마스터 전류 스케일(1/16 단계) |
| 픽셀 계조(grayscale) | GRAM 데이터 | 0~15 | 픽셀별 16단계 회색. `draw_text/fill_buffer`의 `brightness` 인자 |

최종 밝기 ≈ `픽셀계조(0~15) × 0xC1전류(0~255) × ((0xC7+1)/16)`

## 현재 설정값 (확정)
| 항목 | 값 | 비고 |
|---|---|---|
| **패널 구동 전류 (0xC1)** | **0x80 (128, 절반)** | `init_display()`에서 `SSD1322_API_set_contrast(0x80)` 호출 |
| 마스터 전류 (0xC7) | 0x0F (15, 최대) | `SSD1322_API_init()` 기본값 유지 |
| **메인 콘텐츠 계조** | **15** | 선량값·단위·NO COUNT/OVERLOAD·데이터모니터·C/F·펌웨어버전·셀프테스트·POST |
| **알람 표시 계조** | **7** | "Alarm/ALARM" 라벨, 알람 임계값 숫자, "uSv/h" 단위 (`display_alarm`) |
| **설정 점멸 자리 계조** | **3** | 알람 설정 모드에서 편집 중 깜빡이는 자리 (`display_alarm`) |
| 화면 클리어 | 계조 0 | `fill_buffer(tx_buf, 0)` |

## 설계 의도
- **수명 연장은 패널 구동 전류(0xC1)로**: 0xFF(최대)→0x80(절반)으로 낮춰 휘도 감쇠/번인을 크게 늦춘다(대략 수명 2~4배 수준, OLED 가속계수 의존).
- **가독성은 픽셀 계조로**: 콘텐츠는 풀 계조(15)로 또렷하게 표시하되, 하드웨어 전류가 절반이라 실제 화면은 적절히 디밍된다.
- **알람 표시(7)·점멸(3)** 은 메인보다 낮은 계조로 두어 큰 선량값을 돋보이게 하고 보조 정보는 은은하게 표시.
- 배경은 항상 검정(점등 픽셀 최소) → 번인에 유리.

## 코드 위치
- 패널 전류: `Core/Src/main.c` → `init_display()` 의 `SSD1322_API_set_contrast(0x80)`
- 계조: `Core/Src/main.c` 의 각 `draw_text(...)`/`fill_buffer(...)` 마지막 인자
  - 메인 표시 함수: `display_dose_low/high`, `display_monitor`, `display_cf`, `display_firm_ver`, `power_on_selftest` → 15
  - 알람: `display_alarm()` → 7, 점멸 자리 3
- API: `MyLib/SSD1322_OLED_lib_NB/SSD1322_API.c`
  - `SSD1322_API_set_contrast(uint8_t)` → `0xC1`, 0~255
  - `SSD1322_API_set_brightness(uint8_t)` → `0xC7`, 0~15 (현재 미사용)

## 추가 조정 가이드
- **더 어둡게/수명 우선**: `0xC1`을 0x60~0x40으로 낮춤(가독성 확인 필요).
- **더 밝게**: `0xC1`을 0xA0~0xC0으로.
- **번인 추가 완화(선택, 미적용)**: 정적 요소(단위/라벨) 마모 분산을 위해 주기적 ±1~2px 픽셀 시프트.
- 변경은 표시 동작에 영향을 주므로 적용 전 확인을 거친다.

## 변경 이력
- (이번) 패널 전류 0xFF→0x80, 메인 계조 15 유지, 알람 7 / 점멸 3 으로 설정 — 수명·가독성 균형.
